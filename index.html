  • Qus # 1: Map, Filter, ForEach and Find what are difference between these four?

    • Answer: Map, Filter, ForEach and Find এই সবগুলোই প্রোগ্রামিংয়ে বহুল ব্যবহৃত higher-order ফাংশন। এই ফাংশনগুলো মূলত এরে (array)-তে পাওয়া ডাটাগুলো অপারেট করার জন্য ব্যবহৃত হয়। নিচে Map, Filter, ForEach এবং Find মধ্যে পার্থক্যের একটি চিত্র তুলে ধরা হল:

    map() : map হচ্ছে একটি এ্যারের প্রতিটি উপাদানের উপর একটি অপারেশন (যোগ, বিয়োগ, গুন, ভাগ, দ্বিগুন, হাফ ... ইত্যাদি) চালিয়ে নতুন একটি এ্যারে দেয়। এটা সব সয়ম এ্যারে দিবে
    উদাহরণ: const numbers = [1, 2, 3, 4];
    const doubled = numbers.map(num => num * 2); // Result: [2, 4, 6, 8]

    filter() : filter হচ্ছে একটি এ্যারেকে একটি শর্তের আন্ডারে এ্যারের সবগুলো উপাদানকে চেক করা হয়। চেক করার সময় যে উপাদানগুলো ঐ শর্তকে ফুলফিল করে সেই উপাদানগুলোকে নিয়ে একটি নতুন এ্যারে দেয় । এটা সব সয়ম এ্যারে দিবে। সিঙ্গাল ভেলু হলেও সেটা এ্যারের মধ্যে দিবে।
    উদাহরণ: const numbers = [1, 2, 3, 4, 5, 6];
    const evens = numbers.filter(num => num % 2 === 0); // Result: [2, 4, 6]

    find() : array.find() ব্যাবহৃত হয় এ্যারের সবগুলো উপাদান থেকে প্রথম উপদানটিকে খুজে বের করে নিয়ে আসতে। যখন একটি এ্যারেতে একটি কন্ডিশনের আলোকে সার্চ করা হয় এবং কন্ডিশনটি এ্যারের যে উপাদানটি ফুলফিল করে, সেই উপাদানটি একটি সিঙ্গাল ভ্যালু হিসেবে রির্টান করবে এবং এ্যারের বাকি উপাদানগুলোকে সে আর খুঁজবে না।
    উদাহরণ: const numbers = [1, 2, 3, 4, 5];
    const even = numbers.find(num => num % 2 === 0); // Result: 2 (the first even number)

    foreach() : array.foreach এটা array.map() এর মতোই কাজ করে কিন্তু এটা কোনোকিছু রিটার্ন করে না। ফাংশনের ভিরেরই একে কনসল করে দেখতে হয়।
    উদাহরণ: const numbers = [1, 2, 3, 4];
    numbers.forEach(num => console.log(num)); // Prints each number

  • Qus # 2: What are the difference between Local-Storage, Session-Storage and Cookies?
  • Answer: Local Storage, Session Storage, and Cookies হচ্ছে ওয়েভ ডেভোলপমেন্টের ৩টি প্রক্রিয়া যার মাধ্যমে ক্লায়েন্টের কিছু ডাটা ব্রাউজারে সংরক্ষণ করা হয়। প্রত্যেকের নিজস্ব উদ্দেশ্য, বৈশিষ্ট্য এবং ব্যবহারের ক্ষেত্রে রয়েছে। এখানে তাদের মধ্যে মূল পার্থক্যগুলির নিচে উল্লেখ করা হল:

    Local Storage : লোকাল স্টোরেজে ব্যবহার করা হয় ব্রাউজারে বেশি পরিমাণে ডাটা সংরক্ষণ করে রাখার জন্য, ডিভাইস বন্ধ করার পরও এই ডাটা গুলো ব্রাউজারে থেকে যায়। এই ডাটাগুলোর একসেস শুধু ওই ব্রাউজারে পাওয়া যায়। অন্য ব্রাউজার বা অন্য ডিভাইসে নতুনভাবে সেভ হওয়ার পর ওই ডাটাগুলো আবার পাওয়া যায়। এর সাইজ লিমিট ব্রাউজার ভেদে ৫এমবি থেকে ১০এমবি হয়। কুকিজ এর থেকে এর সাইজ লিমিট কম। ডাটা গুলো লাইফটাইম সেভ করে রাখা যায়, যতক্ষন পর্যন্ত ইউজার ডিলিট না করবে। ইউজারের সেটিংস, ইউজার প্রিফারেন্স এতে সেভ করে রাখা হয়।

    Session Storage : সেশন স্টোরেজ প্রতি-সেশন ভিত্তিতে ডেটা সঞ্চয় করার জন্য ডিজাইন করা হয়েছে। ডেটা শুধুমাত্র ওয়েভ পেইজের সেশনের সময়কালের জন্য ব্যবহারযোগ্য (ট্যাব বা উইন্ডো বন্ধ না হওয়া পর্যন্ত)। এটাও লোকাল স্টোরেজের মত ব্রাউজার ভেদে এর একসেসে পাওয়া যায়। এক ব্রাউজারের সেশন অন্য ব্রাউজারে ক্ষেত্রে ভিন্ন। কুকিজ এর থেকে এর সাইজ লিমিট কম।

    Cookies : কুকিজ প্রাথমিকভাবে ব্যবহারকারী সনাক্তকরণ এবং প্রমানকরণ, ট্র্যাকিং এবং সেশন পরিচালনার সাথে সম্পর্কিত ডেটার ছোট আকারে সংরক্ষণের জন্য ব্যবহৃত হয়। কুকিগুলি ডোমেন-নির্দিষ্ট এবং একই ডোমেন থেকে ট্যাব/উইন্ডোজ জুড়ে শেয়ার করা যেতে পারে। কুকির বিভিন্ন মেয়াদ শেষ হওয়ার সময় থাকতে পারে, যা বর্তমান সেশনের বাইরেও স্থায়ী সঞ্চয়ের অনুমতি দেয়। কিছু কুকি হল সেশন কুকি, যেগুলি ব্রাউজার বন্ধ হয়ে গেলে মেয়াদ শেষ হয়ে যায়, অন্যদের নির্দিষ্ট মেয়াদ শেষ হওয়ার তারিখ থাকতে পারে। স্থানীয় স্টোরেজ এবং সেশন স্টোরেজের তুলনায় কুকিজের স্টোরেজ ক্ষমতা অনেক কম। এগুলি সাধারণত ব্যবহারকারীর লগইন অবস্থা বজায় রাখা, ব্যবহারকারীর আচরণ ট্র্যাক করা এবং ব্যবহারকারীর পছন্দগুলি সংরক্ষণ করার মতো কাজের জন্য ব্যবহৃত হয়।

  • Qus # 3: What is Event Loop? How does it work?
  • Answer: Event loop (ইভেন্ট লুপ) জাভাস্ক্রিপ্টের একটি সিক্রেট machenism যার মাধ্যমে জাভাস্ক্রিপ্ট single-threaded প্রোগ্রামিং ল্যাঙ্গুয়েজ হওয়া সত্ত্বেও বাহ্যিক ভাবে multi-threaded প্রোগ্রামিং ল্যাঙ্গুয়েজের মত কাজ করে। ইভেন্ট লুপ গভীরভাবে call stack কে পর্যবেক্ষণ করে এবং যদি call stack খালি বা empty থাকে তবে Event queue থেকে Task call stack এ পাঠায় execution সম্পন্ন করার জন্য। আমরা সকলে জানি, জাভাস্ক্রিপ্ট একটি সিঙ্গেল-থ্রেডেড asynchronous প্রোগ্রামিং ল্যাঙ্গুয়েজ । এটি সাধারণত কীভাবে কাজ করে তা হল:

    Event Queue : ইভেন্টগুলি, যেমন ব্যবহারকারীর ইনপুট, নেটওয়ার্ক অনুরোধ, টাইমার এবং অন্যান্য অ্যাসিঙ্ক্রোনাস ক্রিয়াকলাপগুলিকে একটি সারিতে রাখা হয় যাকে ইভেন্ট কিউ বলা হয়।

    Event Loop Initialization : ইভেন্ট লুপ চলতে শুরু করে, এবং এটি একটি ইনফিনিটি লুপে প্রবেশ করে, ক্রমাগত কিউয়ি ইভেন্টগুলির জন্য ইভেন্ট সারি পরীক্ষা করে।

    Event Processing : সারিতে ইভেন্ট থাকলে, ইভেন্ট লুপ একটি ইভেন্টকে ডিকিউ করে এবং এর ধরন নির্ধারণ করে। ইভেন্টের ধরণের উপর নির্ভর করে, ইভেন্ট লুপ উপযুক্ত ইভেন্ট হ্যান্ডলার বা সংশ্লিষ্ট টাস্ক চালাবে।

    Non-Blocking Execution : ইভেন্ট হ্যান্ডলার বা ইভেন্টের সাথে যুক্ত কাজগুলি সাধারণত নন-ব্লক করার জন্য ডিজাইন করা হয়। এর মানে হল যে ইভেন্টটি সম্পূর্ণ হওয়ার জন্য অপেক্ষা করার সময় তারা পুরো প্রোগ্রামের কার্যপ্রণালী বন্ধ করে না। পরিবর্তে, তারা তাদের কাজ সম্পাদন করে, ইভেন্ট লুপকে অন্যান্য ইভেন্টের জন্য চেক করা চালিয়ে যাওয়ার অনুমতি দেয়।

    Completion and Callbacks : একবার একটি ইভেন্ট হ্যান্ডলার বা টাস্ক সম্পূর্ণ হলে, এটি নতুন ইভেন্ট তৈরি করতে পারে বা কলব্যাকগুলি ট্রিগার করতে পারে। এই নতুন ঘটনা ইভেন্ট সারিতে স্থাপন করা হয়.

    Event Loop Iteration : ইভেন্ট লুপ তার পুনরাবৃত্তি অব্যাহত রাখে, আরো ইভেন্টের জন্য ইভেন্ট সারি পরীক্ষা করে। এই প্রক্রিয়াটি অনির্দিষ্টকালের জন্য পুনরাবৃত্তি হয়, নিশ্চিত করে যে ইভেন্টগুলি আসার সাথে সাথে প্রক্রিয়া করা হয়।

    এই অ্যাসিঙ্ক্রোনাস আচরণটি ডাইনামিক অ্যাপ্লিকেশন তৈরির জন্য অত্যন্ত গুরুত্বপূর্ণ, বিশেষ করে যেগুলি ব্যবহারকারীদের সাথে ইন্টারঅ্যাকশন বা ডাটাবেস এবং API-এর মতো বাহ্যিক সংস্থানগুলি জড়িত। একটি ইভেন্ট লুপ ছাড়া, ইভেন্টগুলি সম্পূর্ণ হওয়ার জন্য অপেক্ষা করার সময় প্রোগ্রামগুলি ব্লক করতে পারে, যার ফলে ব্যবহারকারীর খারাপ অভিজ্ঞতা হয়।
